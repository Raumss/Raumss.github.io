使用Jenkins+GitHub+Hexo实现博客自动部署

Posted on 2018-09-19 | Edited on 2018-09-29

每次在本地写完博客（Markdown 文件），都要手动执行指令生成静态html文件，然后提交到GitHub 上。通过使用Jenkins，将以上流程自动化，减少重复性劳动，提高效率，减少人工错误。

背景知识

Jenkins 是用Java 语言开发的持续集成、部署工具，所以要运行在Java 环境下（JRE）。
Jenkins 可以在JRE 中跑，也可以在Docker 中运行。

本文两种方法都尝试一下，先从本地运行Jenkins 开始。

在本地安装运行Jenkins

安装JRE 和tomcat

在树莓派（CentOS 7.4）上安装，需要找JRE 的ARMv7 版本。
JRE8 + Tomcat 8

下载JDK 8 for ARM v7 版本，JDK 中包含了JRE，下载链接，解压出来发现就是 jdk1.8.0_181。
